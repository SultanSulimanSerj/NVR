# Внешняя аутентификация (LDAP / OIDC)

Состояние на 2026-05: локальные пользователи в SQLite и JWT остаются единственным рабочим путём. Эндпоинт `GET /api/v1/auth/external` (только **admin**) возвращает заготовку контракта для UI и интеграторов.

## План интеграции

1. **LDAP**: bind-сервисная учётка, поиск DN по `uid=` или `sAMAccountName=`, проверка пароля simple bind, кэширование неудачных попыток.
2. **OIDC**: authorization code + PKCE, валидация `id_token`/`userinfo`, маппинг `groups` → роли `viewer` / `operator` (админ только локально или через отдельную группу).
3. **Совместимость**: при включении внешнего провайдера сохранить аварийного локального `breakglass` в `settings` и аудит всех входов.

Подробности эксплуатации см. [RUNBOOK.md](RUNBOOK.md) (раздел доступа после появления реализации).
