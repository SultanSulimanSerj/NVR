# Горизонт 2 — аналитика как SKU (Trassir-подобная ценность)

**Связь:** [PLAN_RECORDER_ANALYTICS_LICENSE.md](PLAN_RECORDER_ANALYTICS_LICENSE.md), [LICENSE_TICKET.md](LICENSE_TICKET.md), [TRASSIR_PARITY_ROADMAP.md](TRASSIR_PARITY_ROADMAP.md) §2.4, фаза B.

## Текущее состояние репозитория

- Детектор движения MOG2 + ROI, события в БД, **AnalyticsWorker**, метрики `nvr_analytics_*`.
- Таблицы AI / поиск детекций, проверки лицензии на AI-эндпоинтах ([LicenseGate](../src/core/LicenseGate.cpp), маршруты `Route*`).

## Целевые SKU (примерная матрица)

| Модуль | Смысл | Зависимости | Бит лицензии (см. LICENSE_TICKET) |
|--------|-------|-------------|-----------------------------------|
| Линия / зона | Пересечение, вторжение | Геометрия на кадре, калибровка | по плану продукта |
| Объект (детектор) | ONNX/YOLO-class | OpenVINO или ONNXRuntime, лимит FPS | бит 0 (AI base) |
| Лица | Галерея, enroll | бит 2 | |
| LPR | OCR pipeline | отдельный модуль + юридика | |

## Полевые метрики

- Форма отчёта: дата, камера, сцена, **FP/FN** (ручная разметка или эталон), версия модели, порог confidence.
- Хранение: приложение к [SLO_RUN_LOG_TEMPLATE.md](SLO_RUN_LOG_TEMPLATE.md) или отдельный `docs/field_analytics_runs/`.

## Юридика (РФ и др.)

- Режимы хранения биометрии, сроки, согласия, удаление по запросу субъекта — отдельный юридический чеклист перед LPR/лицами в проде.

## Критерий готовности горизонта 2 (минимум)

1. Один модуль **end-to-end**: кадр (или событие с bbox) → движок → запись в `events` / `ai_detections` → UI → уведомление.
2. Явная связка **`modules_bitmask`** с отказом 403 на UI/API.
3. Один зафиксированный **полевой прогон** с метриками в репозитории (дата, железо).
