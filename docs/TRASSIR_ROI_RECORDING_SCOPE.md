# Scope: запись по ROI / зонам (отложено от глобального режима камеры)

**Статус:** продуктовое решение зафиксировано для горизонта 1 — **не реализуем** в ядре без отдельного эпика на модель данных и UI.

## Текущее поведение (репозиторий)

- На камере один **`recording_mode`**: `continuous` | `motion` | **`hybrid`** (см. API и [CameraPipeline.cpp](../src/camera/CameraPipeline.cpp)).
- **ROI / motion_rois** влияют на детектор движения и события, но **не** переключают независимые политики записи «по зоне».

## Варианты на будущее (выбор продукта)

1. **Политика JSON** на камеру: массив `{roi_id, mode, schedule_ref}` с наследованием глобального расписания.
2. **Только аналитика по ROI** без смены mux: запись как сейчас, маркировка сегментов по зонам из аналитики.

## Критерий «готово»

- Миграция БД, API PATCH камеры, UI редактор, тесты `test_config_yaml` / интеграция пайплайна, документация OpenAPI.

До принятия варианта 1 или 2 задачи **H1-R*** из бэклогов не ставятся в спринт ядра.
