# Горизонт 3 — корпоративный контур и операторский UX

**Связь:** [TRASSIR_POLISH_PHASES.md](TRASSIR_POLISH_PHASES.md) фаза 3, [TRASSIR_PARITY_ROADMAP.md](TRASSIR_PARITY_ROADMAP.md) фаза C, §2.3, §2.6.

## 1. Диагностика и деградация

| Задача | Описание |
|--------|----------|
| История **stream-stats** | Периодический снимок `GET /api/v1/cameras/{id}/stream-stats` в таблицу + API агрегации + график в UI |
| Алерты | Пороги FPS, дропов, lag; правила уведомлений и события в журнале |

## 2. Карта

- Несколько **этажей** (таблица этажей, `plan_image_url` на этаж, фильтр камер).
- **Drag** позиции камеры на плане с PATCH координат.

## 3. Киоск

- Крупные **touch**-таргеты, читаемость с расстояния.
- **a11y**: фокус, контраст (минимальный набор).
- **Режим оператора**: отдельный URL или флаг киоска — только live/архив без админских действий; согласовать с RBAC.

## 4. Макеты live

- **Версии** пресетов (история изменений или semver имени).
- **Пресет по умолчанию** в профиле пользователя.
- Экспорт/импорт JSON макетов.

## 5. Идентификация

- Реализация **LDAP/OIDC** по [SECURITY_EXTERNAL_AUTH.md](SECURITY_EXTERNAL_AUTH.md) поверх текущего stub в `/api/v1/auth/external`.
