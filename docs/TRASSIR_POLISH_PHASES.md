# План доработок: паритет «зрелый VMS» / Trassir OS (фазы)

**Дата:** 2026-05-13  
**Статус:** живой документ. **Фаза 1** считается **закрытой в репозитории** (сервер, Web UI, OpenAPI, выборочные C++-тесты под новые режимы и конфиг). **Фаза 2+** — по приоритету поля и тендеров.

---

## Фаза 1 — «коробка регистратора» (закрыта)

| Тема | Содержание | Статус |
|------|------------|--------|
| **Гибрид записи** | Режим `hybrid`: непрерывная запись по расписанию + детектор движения для маркировки сегментов и событий; API отклоняет создание без `enable_motion` (`hybrid_requires_enable_motion`). | Реализовано |
| **Таймлайн** | `GET /api/v1/archive/timeline` — счётчики событий по бакетам (`event_count`); в UI Archive — две полосы (движение / события) и легенда. | Реализовано |
| **Мультидиск** | `GET /api/v1/archive/roots-health` (operator+) — существование, свободное место, проба записи по корням; в Web UI страница **Хранилище** — карточка здоровья корней для admin/operator. | Реализовано |
| **ONVIF время** | `POST /api/v1/onvif/device-time-sync` (admin) — выставить UTC на устройстве с часов хоста (best-effort); кнопка в форме камеры в UI. | Реализовано |
| **Карта** | `GET/PUT /api/v1/config/map` — `plan_image_url`; страница Map — фон по URL, блок сохранения для admin. | Реализовано |
| **Макеты Live** | Таблица `live_layouts`, `GET/POST/DELETE /api/v1/live-layouts`; киоск **LocalDashboard** — загрузка с сервера, сохранение/удаление вместе с localStorage. | Реализовано |
| **Поле / поддержка** | `GET /api/v1/system/field-bundle` (admin) + `telemetry.opt_in` в ответе; в System — выгрузка JSON для полевого инженера. | Реализовано (MVP) |
| **Документация API** | `docs/openapi.yaml` (v **1.2.10**) и копия в `webui/public/docs/` — перечисленные эндпоинты, `hybrid` в схеме камеры, `event_count` в бакете таймлайна, каталог камер. | Реализовано |

### Проверки (локально / CI)

| Команда | Назначение |
|---------|------------|
| `cmake -S . -B build -DNVR_BUILD_TESTS=ON && cmake --build build -j && ctest --test-dir build --output-on-failure` | Все C++ unit-тесты (нужны зависимости из корневого `CMakeLists.txt`: FFmpeg, OpenCV, CURL, Python+NumPy при `NVR_WITH_PYTHON=ON`, и т.д.). |
| `cd webui && npm ci && npm run build` | TypeScript + production-сборка Vite. |

На машине без установленного **CMake** или без демона **Docker** полный прогон `ctest` здесь не выполнялся; в CI/Linux-стенде используйте строку из таблицы выше.

Дорожная карта разрывов Trassir и следующие горизонты: [TRASSIR_PARITY_ROADMAP.md](TRASSIR_PARITY_ROADMAP.md), [TRASSIR_HORIZON1_BACKLOG.md](TRASSIR_HORIZON1_BACKLOG.md), [TRASSIR_ROI_RECORDING_SCOPE.md](TRASSIR_ROI_RECORDING_SCOPE.md), [TRASSIR_HORIZON2_ANALYTICS.md](TRASSIR_HORIZON2_ANALYTICS.md), [TRASSIR_HORIZON3_CORP_UI.md](TRASSIR_HORIZON3_CORP_UI.md), [ARCHITECTURE_TRASSIR_PLATFORM.md](ARCHITECTURE_TRASSIR_PLATFORM.md).

## Фаза 2 — запись, архив, ONVIF (углубление)

- **Гибрид по зонам:** JSON-политика «ROI → режим/чувствительность» или отдельные расписания на ROI (сложная модель данных + UI).
- **Таймлайн:** типы событий по цветам, клик «событие → клип», синхронизация с экспортом.
- **Мультидиск:** миграция сегментов при смене корня, алерты при `enospc`/I/O ошибках, SLA в документации.
- **ONVIF:** стабильный PullPoint по вендорам (повторы, digest, разные namespace), синхронизация профилей Media/Imaging, **каталог камер** (CSV/JSON + UI).

---

## Фаза 3 — диагностика и операторский контур

- **История деградации:** периодическое снятие снимка `stream-stats` + алерты при порогах (таблица + API + UI).
- **Карта:** несколько этажей, привязка фона к этажу, drag камер на плане.
- **Киоск:** крупные touch-таргеты, a11y, отдельный «режим оператора» (ограниченное меню, без админских действий).
- **Макеты:** версии, «пресет по умолчанию» в профиле пользователя, экспорт/импорт JSON.

---

## Фаза 4 — коммерция и поле

- **SKU / активация:** связка с центральным сервером лицензий (вне этого репозитория) или расширение офлайн-подписи.
- **Телеметрия:** при `telemetry.opt_in=true` — безопасный снимок (версии, счётчики, без URL/секретов) + канал отправки (sFTP, signed webhook) — отдельный эпик.
- **Обновления / откат:** политика пакетов, `nvr_update.sh` + smoke, документированный rollback.

---

## Зависимости и риски

- **ONVIF SetSystemDateAndTime:** часть камер требует NTP или отклоняет UTC-ручную установку — в UI предупреждение «best effort».
- **Гибрид без motion:** API возвращает `400 hybrid_requires_enable_motion`; при отключённом детекторе на уже настроенной камере пайплайн пишет предупреждение в лог (сегменты без маркировки движением).
- **Поле:** любые SKU/облако — отдельная продуктовая и юридическая проработка.
