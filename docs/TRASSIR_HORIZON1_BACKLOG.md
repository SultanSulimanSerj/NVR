**Связь:** [TRASSIR_POLISH_PHASES.md](TRASSIR_POLISH_PHASES.md) фаза 2, [TRASSIR_PARITY_ROADMAP.md](TRASSIR_PARITY_ROADMAP.md) §2.1–2.2, [PRE_FIELD_AUDIT_2026-05-13.md](PRE_FIELD_AUDIT_2026-05-13.md).

Цель горизонта 1: закрыть ощутимые разрывы **запись / архив / ONVIF / эксплуатация** без перехода к платформенной программе (HA, облако).

---

## 1. Архив и мультидиск

| ID | Задача | Критерий готовности |
|----|--------|---------------------|
| H1-A1 | **Миграция сегментов** при смене `root_path` / извлечении диска | Документированная процедура + утилита или флаг «read-only старый корень»; нет потери ссылок в БД без явного шага админа |
| H1-A2 | **Алерты** при `enospc`, росте I/O latency, `roots-health.writable=false` | Событие в `events` или отдельная таблица + правило уведомлений; дашборд в UI (минимум — список в System) |
| H1-A3 | **SLA в документации** | [SLO.md](SLO.md) и [RUNBOOK.md](RUNBOOK.md) дополнены порогами и действиями при деградации архива |

## 2. Таймлайн и «событие → клип»

| ID | Задача | Критерий готовности |
|----|--------|---------------------|
| H1-T1 | **Цвета/легенда по типам** событий на таймлайне | Отдельная серия или подписи; конфигурируемый маппинг тип → цвет |
| H1-T2 | **Прямой переход** из события с `clip_path` к воспроизведению клипа | Без ручного «экспорт диапазона», если клип уже на диске |
| H1-T3 | Связка **таймлайн → экспорт** одной кнопкой после клика | Уже частично: клик по бакету задаёт `from`/`to`; дальше — явная кнопка «экспорт этого окна» в зоне видимости (UX) |

## 3. ONVIF

| ID | Задача | Критерий готовности |
|----|--------|---------------------|
| H1-O1 | **Повторы SOAP** для PullPoint (реализовано в ядре) | Регресс по [ONVIF_PULLPOINT_CAMERA_MATRIX.md](ONVIF_PULLPOINT_CAMERA_MATRIX.md) |
| H1-O2 | **Digest / WS-Security** варианты для «капризных» камер | Матрица вендоров обновлена |
