# Офлайн-лицензия NVR (билет Ed25519)

Формат версии **1**. Используется libsodium `crypto_sign` (Ed25519): подпись отсоединённая (`crypto_sign_detached`), сообщение — каноническая UTF-8 строка JSON **payload** (без поля `sig`).

## Файлы на регистраторе

| Файл | Назначение |
|------|------------|
| `/etc/nvr-prototype/license.pub` | Сырой публичный ключ, **32 байта** (бинарный). Если отсутствует, подписанный билет проверить нельзя — действует только trial. |
| `/etc/nvr-prototype/license.lic` | Один JSON-объект UTF-8 (см. ниже). Если отсутствует или невалиден — **trial** (лимит каналов из конфига, по умолчанию 8). |

Пути переопределяются в `nvr.yaml` секцией `license:`.

## Fingerprint (`fp`)

Строка из **64 шестнадцатеричных символов** (256 бит): `crypto_generichash` BLAKE2b по сообщению  
`machine_id_trimmed + "|" + first_non_loopback_mac_lower`  
где `machine_id` читается из `/etc/machine-id`, MAC — первый интерфейс в `/sys/class/net/*` кроме `lo`. На не-Linux (сборка без целевой ОС) используется константа `nonlinux-dev` (только для разработки).

Подписанный билет действителен только если поле `payload.fp` **точно совпадает** с fingerprint текущей машины.

## Payload (объект внутри `license.lic`)

Поля (имена фиксированы):

| Поле | Тип | Описание |
|------|-----|----------|
| `fp` | string | Fingerprint машины (64 hex). |
| `max_ch` | int | Максимум камер (>=1). |
| `mod` | int | Битовая маска модулей аналитики (зарезервировано; 0 = только базовые функции). |
| `exp` | int64 | Unix-время истечения; **0** = без срока. |

## Каноническое сообщение для подписи

Строка JSON в **строгом порядке ключей** и без пробелов:

```json
{"exp":<int64>,"fp":"<string>","max_ch":<int>,"mod":<uint>}
```

Числа без кавычек; `fp` — JSON-строка с экранированием по стандарту (если когда-либо понадобятся спецсимволы; для hex fp экранирование не требуется).

Подпись: `crypto_sign_detached` над байтами этой UTF-8 строки.

## Файл `license.lic`

Один JSON:

```json
{
  "payload": { "fp": "…64 hex…", "max_ch": 32, "mod": 0, "exp": 0 },
  "sig": "<base64 standard, без переносов, 64 байта подписи>"
}

```

`sig` — base64 от **сырых 64 байт** подписи Ed25519.

## Режимы на демоне

| Ситуация | Режим API `mode` | Лимит каналов |
|----------|------------------|---------------|
| Нет `license.lic` или пустой | `trial` | `trial_max_channels` из YAML (по умолчанию 8) |
| Есть билет, но нет `license.pub` | `trial` (подпись не проверяется) | trial |
| Подпись/EOL/`fp` не совпали | `degraded` | trial |
| Всё верно | `licensed` | `max_ch` из payload |

## Битовая маска `mod` (резерв)

| Бит | Значение (план) |
|-----|-----------------|
| 0 | Пересечение линии / зона |
| 1 | LPR |
| 2 | Распознавание лиц |

Включение в API/UI модулей — в отдельных задачах (L4 плана продуктов).

**Реализация L4 в API (репозиторий):** для режимов `trial` / `degraded` и при отсутствии нужного бита в подписанном `mod` эндпоинты возвращают **403** с телом `{"error":"license_mod_required","bit":N}`: `GET`/`POST`/`PATCH` под `/api/v1/ai/models` — бит **0**; `GET /api/v1/ai/faces` — бит **2**; `GET /api/v1/events/search` — бит **0** для типов объектов кроме `face` и **`motion`** (поиск по `object=motion` разрешён без бита 0 — строки MOG2 из регистратора), для `object=face` — бит **2**.

## Утилита `nvr_license_sign`

Собирается с `-DNVR_BUILD_LICENSE_TOOL=ON`. **Не** поставляется в публичный `.deb` по умолчанию.

Пример:

```text
nvr_license_sign --secret /secure/nvr_license_sk.key \
  --fp <64-hex от клиента> --max-ch 32 --mod 0 --exp 0 \
  --out license.lic
```

Файл `--secret`: **64 байта** сырого секретного ключа Ed25519 (`crypto_sign_SECRETKEYBYTES`), права на файл только у оператора подписи.

Генерация пары ключей (один раз, на машине подписи):

```bash
# пример через Python + PyNaCl / libsodium — см. внутреннюю инструкцию CI
```

Публичный ключ 32 байта положить в образ/пакет как `license.pub` на регистраторы.

## Безопасность

- Приватный ключ подписи никогда не хранить на регистраторе.
- Публичный ключ можно распространять в `.deb`; смена ключа — новый `license.pub` + перевыпуск билетов.
